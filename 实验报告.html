一、实验目的

通过本次实验，熟练运用 Java 语言、Web前端技术（HTML、JavaScript、CSS）、Web后端技术（Servlet、JSP、Listener、Filter、JDBC等）、MySQL数据库以及框架技术（Spring、Vue3、MyBatis），开发一套基本完善的基于Web的《电商平台》业务系统。

掌握Web开发的基本原理、开发方法及开发流程，掌握运用Web基本技术解决实际问题的开发能力。具体包括：

  1. 掌握前后端分离开发模式，使用 Vue3 + Element Plus 构建用户界面，Spring Boot 构建 RESTful API 后端服务。
  2. 掌握微服务架构设计思想，将系统拆分为用户服务、商品服务、订单服务等独立模块，并通过 API 网关统一入口。
  3. 掌握 Redis 缓存技术在热点数据访问优化中的应用。
  4. 掌握 Docker 容器化部署技术，实现开发、测试与生产环境的一致性。
  5. 掌握 Shiro + Spring Security + JWT 双重认证鉴权机制。
  6. 掌握 WebSocket 实时通信技术在在线客服场景中的应用。

二、实验内容

2.1 基本内容——客户端功能

（1）商品模块：实现商品分类浏览、商品详情展示（图片、名称、价格、库存、规格参数、详情介绍）、商品搜索（关键词模糊搜索、按价格/销量/新品排序）、商品收藏/取消收藏及收藏列表、商品评价（查看评价、提交评价、上传评价图片）。

四、实验过程与分析

4.1 系统架构设计

本系统采用前后端分离 + 微服务架构。前端使用 Vue3 + Element Plus 构建 SPA 单页应用，通过 Axios 与后端 API 通信。后端拆分为四个独立微服务，通过 API 网关统一路由。各服务共享 ecommerce-common 公共模块（实体类、DTO、Mapper、安全组件），数据库层面共享同一 MySQL 实例的不同业务表。

4.5 后端关键实现

4.5.1 JWT + Shiro 双重认证

Spring Security 作为主安全框架，通过 JwtAuthenticationFilter 从请求头 Authorization: Bearer {token} 中解析 JWT，提取 userId 存入 SecurityContextHolder。Shiro 配置 EcommerceRealm 实现细粒度权限控制：SUPER_ADMIN 拥有全部权限（*），ADMIN 拥有 admin 模块权限，USER 拥有用户权限。

4.5.2 API 网关路由

GatewayController 拦截所有 /api/** 请求，根据路径前缀判断目标服务：/api/auth/ → user-service，/api/client/product/ → product-service，/api/client/order/ → order-service 等。通过 RestTemplate 转发请求，复制原始请求头和请求体，返回响应。

4.5.3 Redis 缓存策略

  • 商品详情缓存：使用 product:detail:{productId} 作为 Key，缓存 30 分钟。商品更新时主动清除缓存。
  • Token 会话缓存：使用 user:token:{userId} 存储 Token，普通登录 1 天过期，记住密码 7 天过期。
  • 验证码缓存：使用 verify:code:{email} 存储 6 位数字验证码，5 分钟过期，验证后立即删除。

4.5.4 Docker 多服务编排

docker-compose.yml 定义 6 个服务容器：mysql（健康检查）、redis、user-service、product-service、order-service、gateway-service。各服务通过自定义 ecommerce-net bridge 网络通信，gateway 依赖三个业务服务，业务服务依赖 mysql 和 redis 健康检查通过。Dockerfile 通过 ARG JAR_FILE 支持多服务共用构建。

五、实验创新点

5.1 微服务架构与 API 网关

将传统单体应用拆分为用户、商品、订单三个独立微服务，通过自定义 API 网关实现统一入口和智能路由。各服务独立部署、独立扩展，服务间通过 RestTemplate 进行同步 HTTP 通信。公共模块（ecommerce-common）统一管理实体类、DTO、Mapper、安全组件、WebSocket 等共享代码，避免重复开发。

5.2 Redis 多场景缓存优化

不局限于简单的 KV 缓存，而是针对不同业务场景设计差异化缓存策略：商品详情采用读多写少场景的 30 分钟热缓存 + 主动失效；Token 会话根据"记住密码"选项动态调整过期时间（1 天 / 7 天）；验证码采用 5 分钟短过期 + 一次性消费模式，保障安全性。

5.3 Shiro + Spring Security 双重认证鉴权

创新性地结合 Spring Security（负责 HTTP 层面的 JWT 过滤器链）和 Apache Shiro（负责方法级权限注解和角色权限字符串匹配），实现 HTTP 层 + 业务层的双层安全防护。Shiro Realm 根据用户角色动态分配权限字符串，SUPER_ADMIN 拥有通配符权限。

5.4 Docker 全容器化多服务编排

通过单一 Dockerfile + ARG 参数实现多服务复用构建，docker-compose 统一编排 6 个容器（MySQL、Redis、3 个微服务、Gateway），使用自定义 bridge 网络实现服务间通信，配置健康检查依赖链确保启动顺序正确。

5.5 下单流程中的优惠券实时折扣计算

优惠券折扣在订单创建时实时计算：前端选择优惠券后展示预估折扣，后端创建订单时验证优惠券有效性（存在且启用），折扣金额不超过商品总额，最终实付金额 = 总额 - 折扣。订单记录中保存 totalAmount、discountAmount、payAmount 三个金额字段，便于财务核算。

5.6 完整的前后端工程化实践

前端：Vite 开发服务器 + 代理配置、Pinia 状态管理、Vue Router 路由守卫（鉴权 + 角色校验）、Axios 拦截器（统一 Token 注入 + 错误处理）、ECharts 数据可视化、Element Plus 中文国际化。后端：Spring Boot 多 profile（dev/docker）、Knife4j 自动 API 文档、MyBatis-Plus 逻辑删除 + 自动填充、全局异常处理器、CORS 跨域配置。

六、实验总结

通过本次《Web开发技术》大作业，我完整地经历了一个 Web 电商平台从需求分析、系统设计、编码实现到容器化部署的全流程开发实践。

主要收获：

  1. 深入理解了前后端分离的开发模式，掌握了 Vue3 组合式 API、Element Plus 组件库、Pinia 状态管理等现代前端技术栈。
  2. 掌握了 Spring Boot 3 的微服务拆分方法，理解了 API 网关在微服务架构中的核心作用——统一入口、路由分发、请求转发。
  3. 学会了 Redis 在真实业务场景中的应用，包括缓存策略设计、过期时间管理、主动失效机制等。
  4. 实践了 Docker 容器化技术，理解了 Dockerfile 多阶段构建参数化、docker-compose 多服务编排、网络配置和健康检查等概念。
  5. 掌握了 Shiro + Spring Security + JWT 的双重认证鉴权架构，理解了认证（你是谁）和鉴权（你能做什么）的区别与协作。
  6. 学会了 WebSocket 实时通信协议的应用，实现了简单的在线客服功能。

遇到的问题与解决：

  1. 微服务跨模块引用问题：拆分后 Controller 引用了旧 Service 接口导致编译失败。解决方案是将各 Controller 改为直接注入 Mapper 操作数据库，或调用同服务内的 Service。
  2. Gateway 路由冲突：多个前缀匹配同一路径。解决方案是优化路由规则，确保每个路径前缀唯一映射到一个目标服务。
  3. WebSocket 跨服务问题：微服务下 WebSocket 无法通过 HTTP 网关代理。解决方案是前端直连 product-service 的 WebSocket 端口。

不足与改进方向：目前服务间通信采用同步 RestTemplate，高并发场景下可考虑引入消息队列（如 RabbitMQ）实现异步解耦；可进一步引入 Nacos 实现服务注册与发现，替代硬编码的服务地址配置。

本次实验让我对 Web 开发有了更全面、更深入的认识，从最初只会写简单的 CRUD 接口，到现在能够设计并实现一个架构合理、功能完善的微服务电商系统，这离不开课程中所学知识的积累和实践中的不断探索。
