《Web开发技术》实验（大作业）指导书
实验（大作业）基本信息
Web开发技术课程的实验总学时为八学时，共安排两次实验。两次需要完成《电商平台》开发。第一次实验内容为前端工程及后端工程开发，第二次实验内容为联合调试和系统测试。两次实验需撰写一份实验报告，报告内容包括实验目的、内容、环境、过程与分析、实验创新点、实验总结，详细内容及格式参考报告模板。
实验（大作业）提交成果物
指定提交截止日前，同学需将项目工程压缩包（只打包源码及配置文件，总大小不超过50M）和实验报告分别上传到QQ群中发布的提交链接上。压缩包文件名格式：学号-姓名-班级-大作业-工程压缩包.zip；报告文件名格式：学号-姓名-班级-大作业-实验报告.docx。
实验（大作业）目的
	学生能在实验中，熟练运用 Java 语言、Web前端技术（HTML、JavaScript、CSS）、Web后端技术（Servlet、JSP、Listener、Filter、JDBC等）、MySQL数据库以及框架技术（Spring、VUE3、MyBatis），开发一套基本完善的基于Web的业务系统。通过实验，学生能够掌握Web开发的基本原理、开发方法及开发流程，掌握运用Web基本技术解决实际问题的开发能力。
实验（大作业）内容
	开发基于Web的《电商平台》，具体要求如下：
基本要求：
1.1客户端（电商平台用户使用）功能要求：
面向普通消费者，核心满足浏览、购买、个人中心全流程操作
1. 商品模块
商品分类浏览（按分类筛选商品列表）
商品详情页（商品图片、名称、价格、库存、参数、详情介绍）
商品搜索（精准搜索、模糊搜索、排序：价格 / 销量 / 新品）
商品收藏（收藏 / 取消收藏、收藏列表查看）
商品评价（查看评价、提交评价、上传评价图片）
2. 购物车模块
添加商品到购物车（选择规格、数量）
购物车列表展示（商品图片、名称、单价、数量、小计）
购物车操作（修改数量、删除商品、勾选商品、全选 / 反选）
购物车结算（跳转订单确认页）
3. 订单模块
订单确认（选择收货地址、商品清单、总价计算）
订单提交（生成订单号、选择支付方式）
订单支付（模拟支付 / 对接支付接口）
我的订单（全部 / 待支付 / 待发货 / 待收货 / 已完成 / 已取消订单）
订单操作（取消订单、确认收货、申请退款、查看物流）
1.2管理后台（管理员端）功能
面向平台管理员，核心满足商品、订单、用户、数据全维度管理
1. 管理员认证模块
管理员登录（账号密码、权限验证）
管理员退出
（可选）角色管理（超级管理员 / 普通管理员，权限分配）
2. 首页数据看板
核心数据统计（总用户数、总订单数、总销售额、今日订单 / 销售额）
数据图表（销量趋势、热销商品排行、订单状态统计）
3. 用户管理模块
用户列表（查看所有注册用户信息）
用户搜索（按账号 / 手机号 / 昵称搜索）
用户操作（禁用 / 启用用户、查看用户详情）
4. 商品管理模块
商品分类管理（新增 / 修改 / 删除 / 排序分类）
商品列表（查看所有商品、搜索商品、筛选商品）
商品新增（上传商品图片、填写名称、价格、库存、分类、详情）
商品编辑（修改商品信息、上下架商品、调整库存 / 价格）
商品删除（逻辑删除 / 物理删除）
商品评价管理（查看用户评价、删除违规评价）
5. 订单管理模块
订单列表（查看所有订单、按订单号 / 用户 / 状态筛选）
订单详情（查看商品、地址、支付信息、订单状态）
订单操作（发货、取消订单、处理退款、修改订单状态）
订单导出（导出订单数据为 Excel）
进阶要求：
2.1技术进阶要求：需运用Element Plus、Pinia技术，实现认证鉴权功能（可选用OAuth或Shiro框架）。
2.2界面设计要求：界面需满足简洁美观、操作流畅、布局统一的要求；客户端重点突出商品展示与购买流程，管理后台侧重提升管理效率；所有界面需有明确操作反馈，杜绝布局错乱、功能失效等问题。
2.3客户端进阶要求：
1. 用户认证模块
用户注册（邮箱注册、验证码验证）
用户登录（账号密码登录、记住密码、退出登录）
找回密码（邮箱验证、重置密码）
个人信息修改（头像、昵称、邮箱、密码修改）
2. 首页模块
轮播广告图展示
分类导航栏（一级 / 二级商品分类）
热门商品、新品推荐、促销商品展示
搜索框（关键词搜索、热门搜索词）
3. 收货地址模块
	新增收货地址（收货人、手机号、省市区、详细地址）
	地址列表展示、修改、删除、设为默认地址
4. 商品规格选择（颜色、尺寸）、优惠券功能、秒杀 / 促销活动
5. 其他功能
	客服咨询（简单留言 / 在线聊天）
	系统公告、活动通知展示
	用户反馈（提交问题 / 建议）
2.4管理后台进阶要求：
1. 轮播 / 广告管理
轮播图列表（查看、搜索）
新增 / 编辑 / 删除轮播图（上传图片、设置链接、排序）
2. 系统管理模块
公告管理（发布 / 编辑 / 删除平台公告）
反馈管理（查看用户反馈、回复反馈、标记已处理）
个人中心（修改管理员密码、个人信息）
 3. 数据统计导出、商品批量导入 / 导出、权限分级管理
2.5其他进阶要求：
1. 响应式布局（适配手机 / 电脑）
2. 图片上传
3. 数据分页
拓展要求：
3.1缓存优化：引入 Redis 缓存热点数据，包含商品信息、用户会话等，有效提升页	面加载响应速度。
3.2架构拆分：采用微服务架构，对商品、订单、用户等核心业务模块进行拆分，实现	各模块独立部署、服务间高效通信。
3.3容器化部署：基于 Docker 容器封装项目运行环境，统一开发、测试、生产环境配	置，保障多环境部署一致性。

数据库：MySQL
操作系统：不限
